Mini Project: Build a Custom Developer Ubuntu Image

Follow these terminal commands to compile your customized image and explore the compiled history layers on your machine.

$ docker build -t custom-ubuntu ./week-1/day-3
Explanation: Reads the Dockerfile step-by-step. Compiles your custom packages (git, htop, curl, neofetch) and packs the welcome script.
$ docker run -it --name ubuntu-dev custom-ubuntu
Explanation: Launches the container in active interactive foreground mode. You will immediately see the customized developer hub banner!
$ docker history custom-ubuntu
Explanation: Prints out the compiled filesystem layers, showing the size contribution of each instruction (e.g. FROM, RUN, COPY) exactly as visualized above!

⚠️ Common Image Build FAQs & Troubleshooting

Q1: Why is a .dockerignore file necessary during a build? ▼

Without a .dockerignore, Docker copies the entire directory contents (including temporary files, large assets, or local git files) to the Docker daemon build context. This causes slow, bloated compilations. Excluding them keeps builds instant and lightweight!

Q2: What causes a build cache invalidation? ▼

If any instruction in the Dockerfile is edited, or if any copied file (such as welcome.sh) has changed, Docker invalidates the cache for that specific layer **and all subsequent layers** stacked on top of it, forcing a fresh recompilation.
