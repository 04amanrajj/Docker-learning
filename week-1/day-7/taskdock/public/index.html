Named volumes preserve databases across complete container destructions. Learn how to verify recovery locally:

💥 Disaster Recovery Simulation

×

Follow these quick instructions in your shell terminal to test data persistence:

1. Verify current database records exist:
$ curl -s http://localhost:8080/api/tasks
2. Terminate & delete the container permanently:
$ docker rm -f taskdock-app
3. Confirm host filesystem database integrity:
$ docker volume inspect taskdock-vol
4. Spin up a brand new container instance:
$ docker run -d --name taskdock-app --env-file .env -v taskdock-vol:/data -p 8080:3000 taskdock

✓ Refresh your web browser page. Your tasks will load completely intact, recovered from the named volume mount!
