⚠️ Docker Networks FAQ & Architecture Notes

Q1: Why doesn't the default bridge network support container hostname pings?
The default bridge is a legacy framework. To resolve hostnames automatically, Docker requires a dynamic DNS server module, which is **only instantiated inside user-defined custom networks**. Custom bridges resolve host names to private IPs on the fly.
Q2: Is it safe to expose Redis / database ports on the host system (-p 6379:6379)?
**Absolutely not!** This exposes database listening sockets directly to external network scanners. In a user-defined network, the Node API and Redis can talk internally over `app-net`. You only expose the Node HTTP port (`-p 8080:3000`) so user traffic is routed, while the database remains perfectly isolated.
Q3: What happens when a container dynamically disconnects from a bridge network?
Docker's virtual network switch deletes the container's network interface card (veth). The DNS server removes its entry immediately, and socket attempts to its hostname instantly fail with `ENOTFOUND`, shielding it from other containers.
